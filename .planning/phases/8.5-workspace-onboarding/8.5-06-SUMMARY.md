# Phase 8.5 Plan 06: CLI Package + WorkspaceController Wiring Summary

**One-liner:** WorkspaceController wired to WorkspaceOnboardingService + InitRunner brownfield with DaemonClient pointing to /internal/workspaces/init.

## What Was Built

The final wiring task for Phase 8.5-06: connecting the CLI `bridge init` command to the daemon's internal workspace controller, which in turn calls the WorkspaceOnboardingService (extraction) and attempts InitRunner brownfield mode (AI doc generation, non-fatal).

### WorkspaceController (`packages/nest-core/src/module/workspace/workspace.controller.ts`)

- Route prefix changed from `workspaces` to `internal/workspaces` (per D-05)
- Injects `WorkspaceOnboardingService` for extraction stage (Stage 1)
- Injects `BrainService` as the `ProviderAdapter` for InitRunner (Stage 2)
- Calls `onboardingService.initWorkspace(dto)` ‚Üí returns `{ project, snapshot }`
- Builds `BrownfieldContext` from the snapshot's flat entity fields
- Creates `InitRunner({ projectDir, tools: null, eventStream: null, adapter: brainService })`
- Calls `initRunner.run(...)` ‚Äî failure is caught and logged as non-fatal warning
- Returns `{ success, projectId, slug, snapshotId, vaultDocs }` to CLI

### DaemonClient (`packages/cli/src/client.ts`)

- Renamed from `BridgeClient` to `DaemonClient`
- `POST /internal/workspaces/init` (was `/workspaces/init`)
- `InitWorkspaceRequest` now includes required `projectName` and `slug` fields
- `InitWorkspaceResponse` matches controller response shape

### WorkspaceModule (`packages/nest-core/src/module/workspace/workspace.module.ts`)

- Imports `BrainModule` for `BrainService` injection
- Imports `TypeOrmModule.forFeature([WorkspaceSnapshot, Project])`
- Registers all four workspace services as providers and exports
- Controller wired to the module

### init.ts updates

- Updated to use `DaemonClient` (from `BridgeClient`)
- Derives `projectName` and `projectSlug` from options or workspace path
- Passes required `projectName` and `slug` to `initWorkspace()`

### BrownfieldContext type

- Added to `packages/nest-core/src/module/workspace/types.ts`
- Mirrors `ExtractionSnapshot` shape with stable exported name
- Fields: `repoInfo`, `indexPayload`, `workspacePath`, `projectSlug`

## Known Stubs

**InitRunner `tools` and `eventStream` are `null`**
- **File:** `packages/nest-core/src/module/workspace/workspace.controller.ts` (lines ~48-49)
- **Reason:** `GSDTools` requires local gsd-tools.cjs binary which isn't available in daemon context. `GSDEventStream` would need a no-op implementation. Per plan, InitRunner failure is intentionally non-fatal ‚Äî extraction snapshot is the baseline output.
- **Resolving plan:** Phase 8.6 or later ‚Äî either create minimal GSDTools adapter for daemon context, or accept that AI doc generation is always skipped in daemon mode (extraction-only baseline).
