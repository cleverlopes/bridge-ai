# Phase 8.5 Plan 04: Incremental Sync + Promotion Service Summary

One-liner: **Incremental headSha-based sync for RepoIndexerService and PromotionService with patch/cherry-pick/push strategies enforcing the safe workspace contract.**

## What Was Built

### RepoIndexerService.sync() (Task 1)

Added `sync(projectId: string): Promise<IndexPayload | null>` to the existing `RepoIndexerService`:

- Loads stored snapshot from TypeORM `WorkspaceSnapshot` repository
- Compares `snapshot.headSha` against live `git rev-parse HEAD` via simple-git
- Returns `null` when HEAD unchanged (no re-index needed)
- Calls `bootstrap()`, updates snapshot fields (`headSha`, `indexPayload`, `indexedAt`), persists via `snapshotRepo.save()` when HEAD changed
- Throws `NotFoundException` when no snapshot exists for `projectId`
- Added `@InjectRepository(WorkspaceSnapshot)` to constructor

### PromotionService (Task 2)

New `PromotionService` with three explicit promotion strategies:

**`promoteViaPatch(runId, hostPath)`**
- Runs `git diff HEAD~1..HEAD` on ephemeral workspace to capture run changes
- Throws `BadRequestException('No changes to promote')` if diff is empty
- Writes diff to temp file at `/tmp/bridge-ai-promote-{runId}.patch`
- Applies via `git apply --3way {patchPath}` on host git instance
- Cleanup in `finally` block

**`promoteViaCherryPick(runId, hostPath, commitShas[])`**
- Throws `BadRequestException` if no SHAs provided
- Adds ephemeral workspace path as temp git remote (`bridge-run-{runId[0:8]}`)
- Fetches from temp remote, cherry-picks each SHA into host repo
- Removes temp remote in `finally` block

**`promoteViaPush(runId, remoteName, branchName)`**
- Pushes named branch from ephemeral workspace to the specified remote
- Simplest strategy — used when run workspace already has a tracking branch

### Module update

`workspace.module.ts` adds `PromotionService` to both `providers` and `exports`.

## Safety Invariant Enforced

The daemon **never** writes directly to the host repo. All promotion methods take `hostPath` as an explicit parameter — it is never derived. Ephemeral workspaces at `volumes/runs/{runId}/` are the only write targets during execution; promotion is an explicit operator action.
