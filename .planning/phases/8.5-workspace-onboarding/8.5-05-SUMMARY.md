# Phase 8.5 Plan 05: Safe Workspace Model — Ephemeral Clones, Promotion, Incremental Sync Summary

One-liner: **Complete safe workspace model with per-run ephemeral clones (EphemeralWorkspaceService), three explicit promotion strategies (PromotionService), headSha-based incremental sync (RepoIndexerService.sync), and PipelineService wired to use ephemeral paths for container mounts.**

## What Was Built

This plan completes the safe workspace invariant for bridge-ai: the execution container never sees the host repository directly. All three prior-wave services were already implemented; this plan's verification pass confirmed correctness, confirmed all tests pass, and fixed a broken test setup discovered during the workspace suite run.

### EphemeralWorkspaceService (Task 1 — verified existing)

`EphemeralWorkspaceService` provides per-run isolated workspace clones:

- **`cloneForRun(hostWorkspacePath, runId)`**: Creates `volumes/runs/<runId>/` via `git clone --local --no-hardlinks`. Uses `mkdir(RUNS_BASE, { recursive: true })` to ensure parent exists. Returns the ephemeral run path.
- **`cleanupRun(runId)`**: Removes `volumes/runs/<runId>/` with `rm({ recursive: true, force: true })`. Does not throw if directory doesn't exist.
- **`getRunPath(runId)`**: Utility for callers to compute the path without cloning.

Key safety property: the host workspace path is only read during `cloneForRun()` — it is never passed to the container.

### PromotionService (Task 1 — verified existing)

`PromotionService` implements three explicit promotion strategies. The daemon never writes to the host repo except through these methods, and they all take `hostPath` as an explicit parameter.

**`promoteViaPatch(runId, hostPath)`**
- Runs `runGit.diff(['HEAD~1..HEAD'])` to capture changes from the run
- Throws `BadRequestException('No changes to promote')` if diff is empty
- Writes diff to `/tmp/bridge-ai-promote-{runId}.patch`, applies with `hostGit.raw(['apply', '--3way', patchPath])`
- Cleanup in `finally` block (unlinks temp patch file)

**`promoteViaCherryPick(runId, hostPath, commitShas[])`**
- Throws `BadRequestException` if no SHAs provided
- Adds ephemeral workspace path as temp remote (`bridge-run-{runId[0:8]}`), fetches
- Cherry-picks each SHA individually via `hostGit.raw(['cherry-pick', sha])`
- Removes temp remote in `finally` block

**`promoteViaPush(runId, remoteName, branchName)`**
- Pushes from `runGit.push(remoteName, branchName)` — simplest strategy for run branches with tracking

### RepoIndexerService.sync() (Task 2 — verified existing)

`sync(projectId)` provides incremental indexing:
- Loads `WorkspaceSnapshot` from TypeORM by `projectId`
- Throws `NotFoundException` if no snapshot exists
- Gets live HEAD SHA via `simpleGit(snapshot.workspacePath).revparse(['HEAD'])`
- Returns `null` when `currentHeadSha === snapshot.headSha` (no re-index needed)
- When HEAD changed: calls `bootstrap()`, updates `snapshot.headSha`, `snapshot.indexPayload`, `snapshot.indexedAt`, saves via `snapshotRepo.save()`
- Returns the new `IndexPayload`

### PipelineService Integration (Task 2 — verified existing)

`PipelineService.executeProject()` now conditionally uses ephemeral workspaces:
- Queries `WorkspaceSnapshot` repo by `projectId`
- If snapshot exists **and** `ephemeralWorkspace` is injected: calls `ephemeralWorkspace.cloneForRun(snapshot.workspacePath, planId)` to get isolated run path
- Falls back to `WorkspaceService.provisionWorkspace()` for non-onboarded projects (backward compat)
- Container receives the ephemeral path, never the host repo path
- In `finally` block: calls `ephemeralWorkspace.cleanupRun(runId)` after execution completes or fails

`WorkspaceModule` is now imported in `PipelineModule`, making `EphemeralWorkspaceService` available.

## Safety Invariant Enforced

- Container mounts `volumes/runs/<runId>/` — the ephemeral clone — never the host repo
- Promotion is explicit: operator calls `PromotionService.promoteViaPatch/cherryPick/push()` with the host path as parameter
- `cleanupRun()` runs in `finally` — ephemeral workspace is deleted whether execution succeeds or fails
