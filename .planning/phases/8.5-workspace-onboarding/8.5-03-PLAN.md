2. Implement `workspace-onboarding.service.ts`:

   **validateRepo(workspacePath: string): Promise\<RepoInfo\>**
   - Use `simpleGit(workspacePath)` from `simple-git`
   - Call `git.checkIsRepo(CheckRepoActions.IS_REPO_ROOT)` — throw `BadRequestException` if false
   - Call `git.getRemotes(true)` — find 'origin' or first remote
   - Call `git.status()` for current branch and dirty state
   - Call `git.revparse(['HEAD'])` for headSha
   - Detect baseBranch via `git.raw(['symbolic-ref', '--short', 'refs/remotes/origin/HEAD'])` with 'main' fallback
   - Return `RepoInfo` object

   **validateCredentials(workspacePath: string, repoInfo: RepoInfo, projectId: string): Promise\<void\>**
   - If repoInfo.remoteUrl is null, skip (local-only repo)
   - Call `git.listRemote(['--heads', repoInfo.remoteUrl])` to test read access
   - If SSH URL detected (starts with `git@`): store key via `ksmService.createSecret('workspace-ssh-key-' + projectId, ...)`
   - If HTTPS URL detected: store PAT via `ksmService.createSecret('workspace-https-pat-' + projectId, ...)`
   - Note: For MVP, credentials are optional — validation tests read access only. Credential storage is for when the user provides them explicitly.

   **persistSnapshot(projectId: string, workspacePath: string, repoInfo: RepoInfo, indexPayload: IndexPayload): Promise\<WorkspaceSnapshot\>**
   - Build WorkspaceSnapshot entity from args
   - Use `snapshotRepo.save()` — TypeORM handles upsert when projectId unique constraint matches (use `findOne` + `save` pattern, or `snapshotRepo.upsert()` with `conflictPaths: ['projectId']`)
   - Return saved entity

   **generateVaultDocs(projectSlug: string, snapshot: ExtractionSnapshot): Promise\<string[]\>**
   - Target dir: `resolve(process.cwd(), 'volumes', 'obsidian', 'projects', projectSlug)`
   - Create dir with `mkdir({ recursive: true })`
   - Write 5 files using `writeFile`:
     - `project.md`: frontmatter with name, slug, stack (from manifests), description, created date. Body: "## Overview" with detected stack.
     - `architecture.md`: "## Architecture" with tree summary and entrypoint listing.
     - `stack.md`: "## Technology Stack" — parsed from manifest contents (package.json dependencies, etc.)
     - `decisions.md`: "## Architectural Decision Records" — empty ADR scaffold with frontmatter template.
     - `runbook.md`: "## Runbook" — repo URL, branches, clone instructions, base branch.
   - Return array of written file paths

   **initOnboarding(input: OnboardingInput): Promise\<ExtractionSnapshot\>**
   - If `input.repoUrl` is provided and workspace doesn't contain `.git`, clone it: `simpleGit().clone(input.repoUrl, input.workspacePath)`
   - Call `validateRepo(input.workspacePath)` to get RepoInfo
   - Call `repoIndexer.bootstrap(input.workspacePath, repoInfo)` to get IndexPayload
   - Build ExtractionSnapshot from repoInfo + indexPayload + workspacePath + slug
   - Create project via ProjectService or directly (use slug as project name for MVP)
   - Call `persistSnapshot(projectId, workspacePath, repoInfo, indexPayload)`
   - Call `generateVaultDocs(input.slug, snapshot)`
   - Return ExtractionSnapshot
