## What Was Built

### EphemeralWorkspaceService

Located at `packages/nest-core/src/module/workspace/ephemeral-workspace.service.ts`:

- `cloneForRun(hostWorkspacePath, runId)`: Creates `volumes/runs/<runId>/` directory and clones the host repo with `git clone --local --no-hardlinks`. Returns the absolute ephemeral run path.
- `cleanupRun(runId)`: Removes the run directory with `rm --recursive --force`. Does not throw if the directory is missing.
- `getRunPath(runId)`: Returns `resolve(process.cwd(), 'volumes', 'runs', runId)`.

All three methods are covered by 6 passing tests in the spec file.

### PipelineService Changes

`executeProject()` now:
1. Looks up a `WorkspaceSnapshot` for the current project (`snapshotRepo.findOne({ where: { projectId } })`)
2. If a snapshot exists **and** `EphemeralWorkspaceService` is injected, sets `runId = planId` and calls `cloneForRun(snapshot.workspacePath, runId)` — the ephemeral path goes to `DockerService.createContainer()`
3. Falls back to `WorkspaceService.provisionWorkspace()` when no snapshot exists (backward compatible)
4. In the `finally` block, calls `cleanupRun(runId)` to remove the ephemeral directory (both success and failure paths)

This enforces ONBOARD-09: the Docker container never mounts the host repository — it always receives the ephemeral clone path.

### Module Wiring

- `WorkspaceModule` now exports `EphemeralWorkspaceService` alongside the previously-added services
- `PipelineModule` imports `WorkspaceModule` and `TypeOrmModule.forFeature([WorkspaceSnapshot])`
